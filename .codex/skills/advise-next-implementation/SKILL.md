# 次の実装を提案する

## 目的

このスキルは、今の学習状況から「次に何を作るとよいか」を提案するためのものです。

プロジェクト内に `docs/ai-guidelines.md` がある場合は、先に読んでください。
この教材では、初学者が仕事で見かけやすい構成や考え方に慣れることを重視します。

育成者の立場で、初心者が無理なく進められて、仕事につながる考え方も学べる順番を選んでください。

## 進め方

1. 現在の状態を確認する
   - 学習ログがあれば読みます。
   - リポジトリ内で作業している場合は、関連ファイルを確認します。
   - すでに動いている機能と、直近で練習した内容を把握します。

2. おすすめを1つ選ぶ
   - いくつも並べすぎず、まず取り組むべき1つを選びます。
   - 今のプロジェクトから自然につながる小さな機能にします。
   - 必要な場合だけ、代替案を1〜2個出します。

3. 実装順を小さく分ける
   - 1つずつ確認できる作業に分けます。
   - 各手順で Laravel のどの層を触るかを書きます。
   - 例: Route、FormRequest、Controller、Service、Repository、Model、Blade、Test。

4. 考え方も伝える
   - その作業で何を学べるかを書きます。
   - 初心者がつまずきやすい点を書きます。
   - 実務ではどのように見かけることが多いかを書きます。
   - 今はやらなくてよいことも書きます。

5. 完了の目安を書く
   - ブラウザで何ができればよいかを書きます。
   - 必要なら確認コマンドも書きます。

## 出力形式

特に指定がなければ、次の形で出力してください。

```markdown
## 次のおすすめ
<おすすめする実装>

## 理由
<なぜ今これをやるとよいか>

## 実装の順番
1. <小さな手順> - <触る層 / 学べる概念>
2. <小さな手順> - <触る層 / 学べる概念>

## 意識するポイント
- <概念>: <実務につながる説明>

## 実務ではどう見かけるか
- <実務でよくある形。ただし会社やチームで違うことも補足する>

## 今はやらなくていいこと
- <避けることと理由>

## 完了の目安
- <ブラウザやコマンドで確認できること>
```

## 提案の方針

- 幅広い選択肢より、まず1つの明確な道筋を優先します。
- 今ある CRUD を自然に広げる実装を優先します。
- 実務の話は「多いです」「よく見かけます」「この教材ではこうします」と表現し、断定しすぎないでください。
- Laravel の Todo アプリなら、次のような実装がよい候補です。
  - 完了 / 未完了の絞り込み
  - バリデーションメッセージの日本語化
  - 期限順の並び替え
  - 簡単な検索
  - ページネーション
  - 更新・削除の Feature テスト
- 初心者が CRUD とテストに慣れるまでは、次のものは急がなくてよいです。
  - 認証
  - Docker
  - 複雑な API 設計
  - Vue / React
  - queue
  - MySQL
- できるだけ1回の学習時間で完了できる大きさにします。
