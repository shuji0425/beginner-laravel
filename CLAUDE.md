# Claude向けルール

このリポジトリは、初学者が AI と一緒に Laravel を学ぶための教材です。

Claude を使う場合も、まず [docs/ai-guidelines.md](docs/ai-guidelines.md) を読んでください。

## 重要な方針

- 初学者向けに、丁寧に説明してください。
- 専門用語を使いすぎないでください。
- 実装だけで終わらず、処理の流れと考え方も説明してください。
- 実務でよく見かける構成や考え方を補足してください。
- ただし、会社やチームによって書き方は変わるため、断定しすぎないでください。

## ブランチの扱い

コードを変更する前に、現在のブランチを確認してください。

```bash
git branch --show-current
```

`main` または `master` にいる場合は、作業用ブランチを作るようにユーザーへ促してください。

```bash
git switch -c practice/todo-update
```

すでに作業用ブランチにいる場合は、そのまま作業して構いません。

実務では、作業用ブランチを作ってから変更することが多いです。
この教材でも、その流れに慣れることを大切にしてください。

## Codex skill について

このプロジェクトには `.codex/skills` があります。

Claude は Codex の skill を直接実行できない場合がありますが、内容は学習支援の方針として参考にしてください。

- 学習ログをまとめる
- 次の実装を提案する
- 初心者向けに変更内容を説明する
